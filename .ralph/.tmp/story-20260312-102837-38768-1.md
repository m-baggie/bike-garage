### US-001: Define colorblind-safe CSS color tokens for badges
Status: in_progress
Depends on: None

Description:
As a developer, I want CSS custom properties for all badge colors defined in one place so the entire app uses a consistent, accessible palette.

Acceptance Criteria:
- [ ] Add to client/src/index.css under :root — condition tokens: --condition-excellent: #1D70B8, --condition-good: #00796B, --condition-fair: #92400E, --condition-poor: #9B1D6A, --condition-unknown: #4B5563
- [ ] Add priority tokens: --priority-1: #B91C1C, --priority-2: #C2410C, --priority-3: #92400E, --priority-4: #065F46, --priority-5: #1D4ED8
- [ ] Add badge text token: --badge-text: #FFFFFF
- [ ] No hardcoded hex colors remain in any badge component — all colors reference these variables
- [ ] npm run lint passes
- [ ] Example: inspecting a 'poor' condition badge in devtools shows background: var(--condition-poor)
- [ ] Negative case: no badge uses a raw hex like #ff0000 or color name like 'red' directly in JSX styles or CSS
