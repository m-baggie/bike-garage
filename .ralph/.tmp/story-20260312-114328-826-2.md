### US-006: Responsive layout refinement — desktop and mobile web
Status: in_progress
Depends on: US-003, US-004

Description:
As a user, I want the app to be fully usable on both desktop (1280px+) and mobile web (375px) so I can use it on the go.

Acceptance Criteria:
- [ ] Desktop (≥1024px): two-column layout — left column: photo with overlays + reanalyze button; right column: part detail panel + scrollable parts table
- [ ] Tablet (768–1023px): same two-column layout, narrower columns
- [ ] Mobile (<768px): single column — photo full width on top, part detail panel (bottom sheet on click), parts table below
- [ ] No horizontal scroll at any breakpoint from 375px to 1920px
- [ ] Touch targets for part overlay regions are at least 44x44px on mobile (overlay region minimum size enforced)
- [ ] Bottom sheet on mobile slides up with animation when a part is selected, dismisses on tap outside
- [ ] Parts table on mobile uses a condensed single-line row format (name + priority badge only, expand on tap for details)
- [ ] npm run lint passes
- [ ] Example: open on iPhone SE (375px) → upload works, results display in single column, tapping a region opens bottom sheet
- [ ] Negative case: rotate device to landscape (667px) → layout switches to two-column mid-width, no overflow
