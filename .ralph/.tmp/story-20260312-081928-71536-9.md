### US-008: [Tree 3 — Frontend] Photo upload form component
Status: in_progress
Depends on: US-007

Description:
As a user, I want to select or drag-and-drop a bike photo on the upload page so I can trigger an analysis.

Acceptance Criteria:
- [ ] UploadPage renders a centered card with a drag-and-drop zone and a file picker fallback
- [ ] Accepted file types restricted to image/jpeg, image/png, image/webp in the input element
- [ ] After selecting a file, a preview thumbnail of the image is shown
- [ ] An 'Analyze Bike' button is enabled only when a file is selected
- [ ] Submitting the form calls POST /api/analyze (via fetch or axios) with multipart/form-data
- [ ] While the request is in flight, button shows 'Analyzing…' and is disabled
- [ ] On success, navigate to /results passing the response data via React Router state
- [ ] On error, display an inline error message: 'Analysis failed. Please try again.'
- [ ] Install axios in /client: `npm install axios`
- [ ] Example: drag a JPEG onto the zone → thumbnail appears → click Analyze → loading state shown
- [ ] Negative case: attempt to upload a .gif → browser file picker rejects it due to accept attribute
