### US-012: [Integration] Desktop-first layout polish and basic styling
Status: in_progress
Depends on: US-011

Description:
As a user, I want the app to look clean and professional on desktop so the results are easy to read and act on.

Acceptance Criteria:
- [ ] App has a consistent color palette and typography (system font stack acceptable)
- [ ] Upload page: centered card, max-width 600px, subtle shadow, clear call to action
- [ ] Results page: two-column layout at ≥1024px (photo + bike info left, parts table right), single column below 768px
- [ ] Parts table has alternating row colors for readability
- [ ] Condition and priority badges are visually distinct and legible
- [ ] Loading states (analysis + pricing) are clearly communicated with a spinner or skeleton
- [ ] No horizontal scroll at 1280px
- [ ] npm run lint passes with no errors after styling changes
- [ ] Example: load results page at 1280px → clean two-column layout with no overflow
- [ ] Negative case: view at 375px (mobile) → layout stacks to single column, no elements overflow viewport
