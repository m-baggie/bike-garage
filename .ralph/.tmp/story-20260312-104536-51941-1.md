### US-001: Update Claude Vision prompt to return bounding box coordinates per part
Status: in_progress
Depends on: None

Description:
As the backend, I want Claude Vision to return normalized bounding box coordinates for each identified part so the frontend can render clickable overlays on the photo.

Acceptance Criteria:
- [ ] Claude prompt updated to request a boundingBox field for each part: { x, y, width, height } normalized 0.0–1.0 relative to image dimensions
- [ ] Prompt instructs: x and y are the top-left corner of the bounding region, width and height are the region dimensions
- [ ] Schema validation updated to accept optional boundingBox on each part (not all parts may have a detectable region)
- [ ] Parts with no detectable region have boundingBox: null
- [ ] POST /api/analyze response now includes boundingBox in each part object
- [ ] Existing npm tests updated to reflect new schema shape
- [ ] Example: Rear Derailleur part → boundingBox: { x: 0.6, y: 0.55, width: 0.12, height: 0.1 }
- [ ] Negative case: part with visible_in_image=false → boundingBox is null
