Status: in_progress
Depends on: None

Description:
As the backend, I want Claude Vision to return repair_action, repair_notes, estimated_cost_min, and estimated_cost_max for each part so the frontend can display actionable repair guidance.

Acceptance Criteria:

- [ ] Prompt includes definitions for each repair_action value and cost estimation guidance
- [ ] Backend schema validation updated: new fields accepted, defaults applied if missing (repair_action='service', repair_notes='', estimated_cost_min=0, estimated_cost_max=0)
- [ ] POST /api/analyze response now includes the four new fields on each part object
- [ ] npm test includes a test verifying the new fields are present in the response shape (or defaults applied)
- [ ] Example: Leather Saddle with fair condition → repair_action: 'service', repair_notes: 'Apply leather conditioner...', estimated_cost_min: 15, estimated_cost_max: 80
- [ ] Negative case: Claude omits repair_action for a part → backend applies default 'service', does not return 502
