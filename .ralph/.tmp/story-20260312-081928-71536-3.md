### US-003: [Tree 1 — Backend] Photo upload endpoint with multer temp storage
Status: in_progress
Depends on: US-002

Description:
As a user, I want to POST a bike photo to the backend so it can be processed by Claude Vision.

Acceptance Criteria:
- [ ] POST /api/analyze accepts multipart/form-data with a field named `image`
- [ ] multer stores the uploaded file in OS temp directory (os.tmpdir())
- [ ] Accepted file types: image/jpeg, image/png, image/webp — reject others with 400
- [ ] Max file size: 10MB — reject larger files with 413 and clear error message
- [ ] File is deleted from temp storage after response is sent (success or error)
- [ ] npm install multer already included in US-001
- [ ] Example: POST with a valid JPEG → file saved to temp, path available in req.file
- [ ] Negative case: POST with a .pdf file → 400 'Unsupported file type'
- [ ] Negative case: POST with a 15MB image → 413 'File too large'
