### US-009: [Tree 3 — Frontend] Analysis results display component
Status: in_progress
Depends on: US-007

Description:
As a user, I want to see my bike's metadata and a detailed parts analysis table so I know what needs attention.

Acceptance Criteria:
- [ ] ResultsPage reads analysis data from React Router location.state
- [ ] Bike metadata section shows: brand, model, type, year, color, frame_material, overall_condition, summary
- [ ] Parts section renders a table with columns: Part Name, Component Group, Brand/Model, Condition (badge), Priority (badge), Condition Notes
- [ ] Condition badges: excellent=green, good=blue, fair=yellow, poor=red
- [ ] Priority badges: 1=red 'Immediate', 2=orange 'Soon', 3=yellow 'Monitor', 4=blue 'OK', 5=green 'New'
- [ ] Parts table is sortable by Priority (default sort: lowest number first = most urgent)
- [ ] If location.state is null or missing, redirect to / with a message 'No analysis data — please upload a photo'
- [ ] Example: results page loaded with 8 parts → table shows all 8 sorted by priority
- [ ] Negative case: results page navigated to directly with no state → redirects to upload page
