### US-002: Update condition badges with colorblind-safe colors and icons
Status: in_progress
Depends on: US-001

Description:
As a user with color vision deficiency, I want condition badges to use distinct colors AND icons so I can tell them apart without relying on color alone.

Acceptance Criteria:
- [ ] Condition badge component updated with new palette using CSS tokens from US-001
- [ ] Each condition value maps to: excellent → Blue (#1D70B8) + '★' icon, good → Teal (#00796B) + '✓' icon, fair → Amber (#92400E) + '●' icon, poor → Magenta (#9B1D6A) + '✕' icon, unknown → Gray (#4B5563) + '?' icon
- [ ] Badge renders as: [icon] [label] — e.g. '✓ Good', '✕ Poor'
- [ ] White (#FFFFFF) text used on all condition badge backgrounds — all combinations pass 4.5:1 contrast
- [ ] Badge shape (pill/rounded) and padding unchanged from Phase 1
- [ ] Simulating deuteranopia in Chrome devtools (Rendering → Emulate vision deficiency → Deuteranopia) shows all 5 conditions as visually distinct
- [ ] npm run lint passes
- [ ] Example: a part with condition 'poor' renders a magenta badge with '✕ Poor' in white text
- [ ] Negative case: condition value 'unknown' renders the gray badge with '? Unknown' — not invisible or unstyled
