### US-003: Update priority badges with colorblind-safe colors and icons
Status: in_progress
Depends on: US-001

Description:
As a user with color vision deficiency, I want priority badges to use distinct colors AND icons so I can immediately understand urgency without relying on color alone.

Acceptance Criteria:
- [ ] Priority badge component updated with new palette using CSS tokens from US-001
- [ ] Each priority level maps to: 1 → #B91C1C + '⚠ Immediate', 2 → #C2410C + '↑ Soon', 3 → #92400E + '● Monitor', 4 → #065F46 + '✓ OK', 5 → #1D4ED8 + '★ New'
- [ ] White (#FFFFFF) text used on all priority badge backgrounds — all combinations pass 4.5:1 contrast
- [ ] Badge renders as [icon] [label] — e.g. '⚠ Immediate', '✓ OK'
- [ ] Badge shape (pill/rounded) and padding unchanged from Phase 1
- [ ] Simulating deuteranopia in Chrome devtools shows all 5 priority levels as visually distinct
- [ ] npm run lint passes
- [ ] Example: priority 1 part renders a dark red badge with '⚠ Immediate' in white text
- [ ] Negative case: priority 5 (New) badge is clearly distinguishable from priority 4 (OK) badge under deuteranopia simulation
