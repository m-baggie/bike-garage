### US-010: [Tree 3 — Frontend] Pricing results display — integrated into results page
Status: in_progress
Depends on: US-009

Description:
As a user, I want to see Performance Bicycle pricing for each identified part so I know what replacements will cost.

Acceptance Criteria:
- [ ] After analysis data loads, ResultsPage automatically calls POST /api/pricing with the parts array
- [ ] While pricing is loading, each part row shows a 'Loading prices…' skeleton or spinner
- [ ] Once pricing loads, each part row expands to show up to 3 Performance Bicycle results: product title, price, availability, and a link to the product page
- [ ] If a part has no pricing results, show 'No results found on Performance Bicycle'
- [ ] Pricing load errors are shown inline per part: 'Pricing unavailable'
- [ ] Layout remains clean and scannable at 1280px width
- [ ] Example: 6-part analysis → pricing call fires → each part row shows 1-3 product options with prices
- [ ] Negative case: pricing API returns empty array for a part → 'No results found on Performance Bicycle' shown for that row
