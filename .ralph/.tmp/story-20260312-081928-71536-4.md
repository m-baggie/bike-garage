### US-004: [Tree 1 — Backend] Claude Vision API integration with structured JSON output
Status: in_progress
Depends on: US-003

Description:
As the backend, I want to send the uploaded image to Claude Vision and receive a validated structured JSON analysis of the bike's parts, condition, and metadata.

Acceptance Criteria:
- [ ] Install Anthropic SDK: `npm install @anthropic-ai/sdk` in /server
- [ ] Image is read from temp file and base64-encoded before sending to Claude
- [ ] Claude is called with model `claude-sonnet-4-5` (or latest vision-capable model)
- [ ] System prompt instructs Claude to return ONLY valid JSON — no markdown, no prose, no code fences
- [ ] Prompt specifies the full schema: bike object (brand, model, type, year, color, frame_material, notes) and parts array (id, name, component_group, brand, model, condition, condition_notes, priority 1-5, priority_label, search_query, visible_in_image)
- [ ] Prompt includes the priority scale (1=Immediate, 2=Soon, 3=Monitor, 4=OK, 5=New) and condition values (excellent/good/fair/poor/unknown)
- [ ] Prompt includes component group taxonomy: Drivetrain, Brakes, Wheels, Handlebars/Cockpit, Saddle/Seatpost, Frame/Fork, Accessories
- [ ] Backend validates response is parseable JSON and contains required top-level keys: bike, parts, overall_condition, summary
- [ ] Invalid or unparseable Claude response returns 502 with error message
- [ ] Example: valid bike photo → Claude returns JSON with bike metadata and 5-10 parts
- [ ] Negative case: Claude returns non-JSON text → backend returns 502 'Analysis failed: invalid response format'
