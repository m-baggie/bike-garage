### US-004: Part detail panel component
Status: in_progress
Depends on: US-003

Description:
As a user, I want a dedicated detail panel that shows full information for the selected part, including its pricing results, so I don't have to scan the full table.

Acceptance Criteria:
- [ ] PartDetailPanel component receives the active part object and its pricing results as props
- [ ] Panel displays: part name (heading), component group, brand and model, condition badge, priority badge, condition notes (full text)
- [ ] Pricing section shows up to 3 Performance Bicycle results: product title (linked), price, availability
- [ ] If pricing results are empty, shows 'No results found on Performance Bicycle'
- [ ] Panel shows a 'No part selected — click a part on the photo' placeholder when no part is active
- [ ] On mobile (< 768px), panel renders as a bottom sheet or modal overlay instead of a side column
- [ ] npm run lint passes
- [ ] Example: active part is a brake caliper with 2 pricing results → panel shows both with prices and links
- [ ] Negative case: active part has no pricing results → panel still renders correctly with the 'no results' message
