### US-002: Replace flat parts table with urgency-grouped sections
Status: in_progress
Depends on: US-001

Description:
As a user, I want parts grouped by urgency so the most critical repairs are immediately visible at the top.

Acceptance Criteria:
- [ ] ResultsPage replaces the single flat parts table with three named sections: 'Action Required' (priority 1-2), 'Monitor' (priority 3), 'Good Shape' (priority 4-5)
- [ ] Each section has a colored header: Action Required=red, Monitor=amber, Good Shape=green
- [ ] Each section is collapsible — chevron toggle, all sections open by default
- [ ] Sections with zero parts are not rendered
- [ ] Within each section, parts are sorted by priority ascending (1 before 2, etc.)
- [ ] Each part row now includes a 'Repair Action' badge: clean_lube='🔧 Clean & Lube' (blue), adjust='🔧 Adjust' (purple), service='🔧 Service' (orange), replace='🔧 Replace' (red)
- [ ] Repair notes displayed below condition notes in an italicized smaller font
- [ ] npm run lint passes
- [ ] Example: 10-part analysis with 2 priority-1, 5 priority-3, 3 priority-4 → three sections rendered with correct part counts
- [ ] Negative case: all parts are priority 4-5 → only 'Good Shape' section renders, 'Action Required' and 'Monitor' sections are hidden
