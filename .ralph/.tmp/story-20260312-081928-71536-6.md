### US-006: [Tree 2 — Scraping] Pricing API endpoint — batch pricing for all parts
Status: in_progress
Depends on: US-005

Description:
As the backend, I want a POST /api/pricing endpoint that accepts an array of parts and returns scraped pricing for each one.

Acceptance Criteria:
- [ ] POST /api/pricing accepts JSON body: { parts: [{ id, name, search_query }] }
- [ ] Scrapes Performance Bicycle for each part using its search_query field
- [ ] Returns JSON: { pricing: [{ part_id, part_name, search_query, results: [{title, price, availability, url}] }] }
- [ ] Parts are scraped sequentially (not concurrently) to avoid hammering the target site
- [ ] Endpoint returns 200 even if all parts have empty pricing results
- [ ] Returns 400 if parts array is missing or empty
- [ ] npm test includes a test verifying POST /api/pricing with valid input returns 200 with correct shape
- [ ] Example: POST with 3 parts → response contains 3 pricing entries (results may be empty arrays)
- [ ] Negative case: POST with missing parts field → 400 'parts array is required'
