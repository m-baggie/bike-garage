### US-002: Render clickable part overlay regions on bike photo
Status: in_progress
Depends on: US-001

Description:
As a user, I want to see subtle clickable regions on the bike photo corresponding to identified parts so I know where to click for details.

Acceptance Criteria:
- [ ] BikeImageOverlay component renders the bike photo with absolutely-positioned div or SVG overlay regions for each part that has a non-null boundingBox
- [ ] Overlay regions are positioned using normalized boundingBox coordinates scaled to the rendered image dimensions
- [ ] On hover, overlay region shows a semi-transparent highlight ring (outline, not fill) so the photo remains visible
- [ ] Cursor changes to pointer over clickable regions
- [ ] Regions with visible_in_image=false or null boundingBox are not rendered
- [ ] Image and overlays scale correctly when the browser window is resized
- [ ] npm run lint passes
- [ ] Example: 8-part analysis with 6 parts having bounding boxes → 6 overlay regions visible on photo
- [ ] Negative case: analysis with 0 bounding boxes → photo renders normally with no overlays, no errors
