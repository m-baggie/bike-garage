### US-005: Reanalysis button — trigger fresh Claude Vision call
Status: in_progress
Depends on: US-001

Description:
As a user, I want a reanalysis button that sends the same uploaded photo back to Claude Vision so I can get a fresh analysis without re-uploading.

Acceptance Criteria:
- [ ] A 'Reanalyze' button is displayed on the Results page above the bike photo
- [ ] Clicking the button POSTs the same image file (held in component state or re-read from a stored File object) to POST /api/analyze
- [ ] During reanalysis, button shows 'Reanalyzing…' and is disabled; a loading overlay covers the photo
- [ ] On success, all analysis state is replaced: parts list, bike metadata, active selection, and pricing results are all cleared and reloaded
- [ ] Pricing is automatically re-fetched after reanalysis completes (same as initial flow)
- [ ] On error, show inline error 'Reanalysis failed. Please try again.' without losing the previous results
- [ ] If the user navigated to results from a fresh upload, the original File object is available in React Router state for reuse
- [ ] npm test includes a test that the reanalysis endpoint call is made with the correct content type
- [ ] Example: click Reanalyze → loading overlay → new parts appear (may differ from first run)
- [ ] Negative case: reanalysis fails due to network error → previous results remain visible with error message shown
