### US-004: Compact collapsible parts list with expand-all toggle
Status: in_progress
Depends on: US-001

Description:
As a user, I want a compact default view of the parts list with the ability to expand individual parts or all at once so I can scan quickly without being overwhelmed.

Acceptance Criteria:
- [ ] Each part row in all urgency sections defaults to a single compact line: part name (font-weight 500) + condition badge + priority badge + repair action badge + a chevron (›) expand button, all on one line
- [ ] Compact row fits on one line without wrapping at 1280px desktop width
- [ ] Clicking the chevron or anywhere on the row expands it to show: condition notes, repair notes (italic), pricing results or estimated cost range
- [ ] Expanded state shows a ‹ collapse chevron
- [ ] Each urgency section header (Action Required / Monitor / Good Shape) has an 'Expand All' link on the right — clicking it expands all rows in that section; changes to 'Collapse All' when all are expanded
- [ ] Sections themselves remain collapsible (existing behavior) — the expand/collapse all only affects rows within an open section
- [ ] No information is removed — all detail from current view is still accessible after expanding
- [ ] npm run lint passes
- [ ] Example: 10-part results page → all rows compact on load → click 'Expand All' on Action Required → 2 rows expand showing full detail
- [ ] Negative case: part with no condition notes or repair notes — expanded state still renders cleanly with just pricing, no empty placeholder text
