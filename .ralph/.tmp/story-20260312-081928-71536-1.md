### US-001: [Setup] Project scaffold — monorepo structure, scripts, and env config
Status: in_progress
Depends on: None

Description:
As a developer, I want the project folder structure, scripts, and environment configuration in place so all three work trees can begin in parallel.

Acceptance Criteria:
- [ ] Root directory contains /client (React/Vite) and /server (Express) folders
- [ ] Root package.json includes scripts: `dev` (starts both client and server), `lint`, `test`
- [ ] server/package.json includes dependencies: express, multer, cors, dotenv, axios, cheerio — install with `npm install express multer cors dotenv axios cheerio`
- [ ] client/package.json scaffolded via Vite: `npm create vite@latest client -- --template react`; install with `npm install`
- [ ] server/.env.example contains ANTHROPIC_API_KEY=your_key_here and CLIENT_URL=http://localhost:5173
- [ ] server/.env is gitignored
- [ ] README.md documents: how to set env vars, how to run `npm run dev` from root
- [ ] Example: cloning repo, copying .env.example to .env, running npm run dev — both client and server start without errors
- [ ] Negative case: missing ANTHROPIC_API_KEY in .env causes server startup to log a clear warning (not a crash)
