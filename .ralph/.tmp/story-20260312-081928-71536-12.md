### US-011: [Integration] End-to-end flow — upload photo, get analysis, show pricing
Status: in_progress
Depends on: US-004, US-006, US-010

Description:
As a user, I want the full workflow to work end-to-end: upload a photo, see Claude's analysis, and see real pricing from Performance Bicycle.

Acceptance Criteria:
- [ ] Upload page successfully POSTs to POST /api/analyze and receives valid analysis JSON
- [ ] POST /api/analyze calls Claude Vision and returns the structured bike/parts JSON
- [ ] ResultsPage calls POST /api/pricing after receiving analysis data
- [ ] Full flow completes without console errors in browser or server
- [ ] Temp image file is cleaned up server-side after analysis
- [ ] Tested manually: upload a real bike photo → full results render with bike metadata, parts table, and pricing
- [ ] Example: upload Trek road bike photo → see 5-10 identified parts with conditions and Performance Bicycle prices
- [ ] Negative case: upload a non-bike photo (e.g. a cat) → Claude still returns valid JSON (may have few/no parts), app renders gracefully without crashing
