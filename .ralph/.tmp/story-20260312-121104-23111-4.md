### US-004: Maintenance summary card at top of results page
Status: in_progress
Depends on: US-001

Description:
As a user, I want a summary card at the top of the results page that shows urgent part counts and total estimated cost so I get an at-a-glance view of my bike's health.

Acceptance Criteria:
- [ ] Summary card renders above the urgency sections, below the bike metadata
- [ ] Card shows four stats in a row: '⚠ X parts need attention' (red, count of priority 1-2), '● X to monitor' (amber, count of priority 3), '✓ X in good shape' (green, count of priority 4-5), '💰 Est. $min – $max total' (neutral, sum of all parts' cost ranges)
- [ ] If all parts have cost 0, the cost stat shows 'Cost unknown'
- [ ] Card has a white background, subtle border, rounded corners, and 16px padding
- [ ] On mobile (<768px) stats wrap to 2x2 grid instead of a single row
- [ ] npm run lint passes
- [ ] Example: 10 parts — 2 urgent, 5 monitor, 3 good, total estimated $120–$340 → card shows all four stats correctly
- [ ] Negative case: 0 urgent parts → '⚠ 0 parts need attention' is hidden; card only shows monitor + good shape + cost stats
