### US-002: [Tree 1 — Backend] Express server with health check and CORS
Status: in_progress
Depends on: US-001

Description:
As a developer, I want a running Express server with CORS configured and a health check endpoint so the frontend can communicate with it.

Acceptance Criteria:
- [ ] Express server starts on PORT from .env (default 3001)
- [ ] CORS configured to allow requests from CLIENT_URL env var
- [ ] GET /api/health returns { status: 'ok', timestamp: ISO-string }
- [ ] Server uses dotenv to load environment variables on startup
- [ ] npm test includes a test that GET /api/health returns 200 with status 'ok'
- [ ] Example: curl http://localhost:3001/api/health → 200 { status: 'ok' }
- [ ] Negative case: request from disallowed origin receives CORS error
