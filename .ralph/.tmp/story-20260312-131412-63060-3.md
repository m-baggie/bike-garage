### US-003: Enlarge bike photo hero and add always-visible priority pins
Status: in_progress
Depends on: US-001

Description:
As a user, I want the bike photo to be the dominant visual element and part pins to always be visible so I immediately understand what to interact with.

Acceptance Criteria:
- [ ] On desktop (≥1024px), left column (photo + overlays) width increased to 55% of the two-column layout
- [ ] Photo container gets box-shadow: 0 8px 32px rgba(0,0,0,0.18) and border-radius: 12px
- [ ] Each part with a boundingBox gets a small circular pin marker at its center (bounding box cx, cy): 12px diameter, filled with the part's priority color, white border 2px, always visible (not just on hover)
- [ ] Pin has a subtle pulse animation on hover: scale(1.3) over 0.2s
- [ ] Pin has a tooltip (HTML title attribute) showing the part name
- [ ] Active part pin scales to 16px and shows a ring around it
- [ ] npm run lint passes
- [ ] Example: results page loads → 8 small colored dots visible on bike photo without any interaction
- [ ] Negative case: parts with null boundingBox render no pin — no dot at 0,0
