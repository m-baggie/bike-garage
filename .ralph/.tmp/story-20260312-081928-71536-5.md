### US-005: [Tree 2 — Scraping] Performance Bicycle part search scraper
Status: in_progress
Depends on: US-001

Description:
As the backend, I want to search Performance Bicycle by part name and scrape real pricing and availability so users see current market prices.

Acceptance Criteria:
- [ ] cheerio is installed in /server (included in US-001 setup)
- [ ] scrapePartPricing(searchQuery) function accepts a search query string and returns an array of up to 3 results
- [ ] Each result contains: { title: string, price: string, availability: string, url: string }
- [ ] HTTP request uses a realistic User-Agent header to avoid simple bot blocks
- [ ] If no results are found, return an empty array (not an error)
- [ ] If the HTTP request fails (network error, non-200 status), return an empty array and log the error
- [ ] Function is exported from server/src/scrapers/performanceBicycle.js
- [ ] npm test includes a test that scrapePartPricing returns an array (may be empty if site unreachable — test shape, not content)
- [ ] Example: scrapePartPricing('Shimano Claris rear derailleur 8-speed') → [{title, price, availability, url}, ...]
- [ ] Negative case: scrapePartPricing('xyzzy nonexistent part 9999') → []
